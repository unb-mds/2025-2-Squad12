# Storymap

Este storymap resume a trajetória principal da pessoa usuária no **É Crypto**, desde o primeiro contato até a exploração dos conteúdos sobre criptomoedas.

---

## Trajetória principal da pessoa usuária

1. A pessoa acessa o site **É Crypto**.
2. Visualiza a página inicial com uma explicação do projeto e um resumo do que é possível fazer no portal.
3. Escolhe navegar pela listagem de conteúdos (notícias, curiosidades, materiais informativos).
4. Utiliza filtros básicos (por exemplo: tipo de conteúdo, criptomoeda, fonte ou período).
5. O sistema executa a busca e exibe um **feed de conteúdos normalizados**, mostrando:
   - título;
   - breve descrição;
   - data;
   - fonte;
   - link para o conteúdo original.
6. A pessoa seleciona um item da lista para ver mais detalhes.
7. A partir do card detalhado, ela é direcionada para o site de origem da notícia/conteúdo.
8. A pessoa retorna ao É Crypto para:
   - continuar navegando;
   - alterar filtros;
   - procurar novos conteúdos relacionados.

---


## Conexão com as Sprints

De forma geral, a trajetória foi construída ao longo das sprints:

- **Sprints iniciais**  
  - definição do tema e visão do produto;
  - entendimento das fontes de dados e do domínio de criptomoedas.

- **Sprints focadas no backend (pipeline de coleta)**  
  - implementação dos spiders de scraping;
  - normalização dos dados coletados.

- **Sprints focadas em experiência de uso**  
  - definição de como o feed de conteúdos deve ser exibido;
  - criação/refino do protótipo no Figma.

- **Sprints de refinamento e entrega final**  
  - melhorias na usabilidade;
  - ajustes na documentação (incluindo este storymap);
  - preparação para demonstração do projeto.
